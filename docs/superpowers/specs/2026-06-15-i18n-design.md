# Internationalisation (vue-i18n) + English

Make every user-facing string translatable and add an English locale alongside
the existing Ukrainian, with an in-app language switcher.

## Library

`vue-i18n` v11, Composition API mode (`legacy: false`, `useI18n`). Vue is 3.5.38.

## What gets translated

- All UI chrome in the `.vue` components.
- The research vocabulary in `researchResults.ts` (class / parameter / filter
  labels + the `describeNode` phrases).
- User-facing strings in stores (`plans.ts`: default plan name, import errors).

NOT translated: research node `name` / `category` — these come from the game
data and stay English in both locales.

## i18n module — `src/i18n/`

- `index.ts` — `createI18n({ legacy: false, locale, fallbackLocale: 'en', messages })`.
  - `missing(locale, key)` handler returns the key's last segment, so an unknown
    research token (e.g. a new game parameter) renders as the raw token instead
    of a key path. This replaces the old `?? raw` fallbacks.
  - Locale resolution on boot: `localStorage['wz-locale']` if set and valid,
    else detect from `navigator.language` (`uk*` → `uk`, otherwise `en`).
  - `setLocale(l)` updates the composer locale, `localStorage`, and
    `<html lang>`.
- `uk.ts`, `en.ts` — message catalogs, mirrored key structure.

Key namespaces: `app`, `topbar`, `search`, `branch.*`, `status.*`, `details.*`,
`node.*`, `research.class.*`, `research.param.*`, `research.filter.*`,
`research.phrase.*`, `plan.*`, `game.*`, `model.*`.

## researchResults.ts refactor

Functions stop owning translation tables. They take a translator:

```
type Translate = (key: string, named?: Record<string, unknown>) => string;
formatResult(result, names, t)
describeNode(node, names, t)
```

- `formatResult` → `t('research.param.' + parameter)` etc. (missing handler gives
  raw-token fallback).
- `describeNode` → `t('research.phrase.unlocks', { items })` /
  `t('research.phrase.improves', { items })` / `t('research.phrase.andMore', { count })`.
- `resolveName`, `minResearchSeconds`, `formatDuration` stay translation-free.
- Tests build a real `uk` i18n instance and pass its `t`, asserting the same
  Ukrainian output as before (so coverage is preserved, not weakened).

## Components

`useI18n()` in each; replace literals with `t(...)`. Move `BRANCH_LABELS` /
`STATUS_LABELS` lookups to `t('branch.' + node.branch)` /
`t('status.' + status)`. `DetailsPanel` passes `t` into the researchResults
helpers.

## Language switcher

A small `uk / en` control in `TopBar`, calling `setLocale`. Persisted choice is
read back on next boot.

## main.ts

`createApp(Root).use(createPinia()).use(i18n).mount('#app')`.

## Out of scope

- Translating game-data node names/categories.
- More than two locales (structure stays open to adding them).
- Pluralisation beyond simple `{count}` interpolation (UK abbreviations kept).

## Verification

- `vue-tsc -b` clean; `vitest` green (researchResults tests now drive `t`).
- `pnpm dev`: switch UK↔EN — chrome, details panel, research upgrades, and
  descriptions all flip; reload keeps the chosen locale; node names stay English.
